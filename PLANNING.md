## 3. 핵심 컨셉: 하루 루틴 기반 학습

사용자가 1주일 루틴을 설정하면, AI가 그 시간에 맞춰 먼저 말을 건다.

**예시 루틴**:
| 시간 | 요일 | 학습 타입 | 예시 |
|------|------|-----------|------|
| 06:00 | 매일 | 인사말 | ¡Buenos días! ¿Cómo dormiste? |
| 08:00 | 평일 | 상황 단어 | 운전/교통 관련 단어 5개 |
| 09:00 | 평일 | 복습 | 전날 배운 내용 퀴즈 |
| 13:00 | 평일 | 새 학습 | 오늘의 단어 + 문법 1가지 |
| 15:00 | 평일 | 복습 | 점심에 배운 것 퀴즈 |
| 17:30 | 평일 | 오답 복습 | 틀린 것만 다시 |
| 21:00 | 매일 | 일기 쓰기 | 오늘 하루를 스페인어로 |

**학습 타입 6가지**:
1. 인사말 — 시간대별 자연스러운 인사
2. 상황 단어 — 맥락(운전, 직장, 식사 등)에 맞는 단어
3. 새 학습 — 단어 + 문법 1가지
4. 복습 — 이전에 배운 것 퀴즈
5. 오답 복습 — 틀린 것만 다시
6. 일기 쓰기 — 자유 작문 + AI 교정

## 4. AI 전략

**결정**: 로컬 LLM (Ollama) 사용
**이유**: 비용 $0, RTX 5090 x2 (64GB VRAM)로 클라우드급 품질 가능

**모델 후보**:
- Llama 3.3 70B (Q4, ~40GB VRAM) — 범용 최강
- Qwen 2.5 72B (Q4, ~40GB VRAM) — 다국어 강점
- Mistral Large 2 (Q4, ~65GB VRAM) — 스페인어 특화

**최종 모델**: 미결정 (설치 후 테스트로 결정)

**API 형식**: OpenAI 호환 (bunny-app과 동일한 패턴)
**엔드포인트**: http://127.0.0.1:11434/v1 (Ollama 기본값)

**Yojal AI의 하네스 (앱 내부)**:
- 인수인계 노트: 사용자의 학습 이력 DB 저장 (배운 단어, 틀린 문제)
- 깔끔한 책상: 최근 N턴만 컨텍스트에 전달 (전체 이력 x)
- 자동 품질 검사: 문법 교정 결과 검증
- 업무 위임: 학습 타입별 전용 프롬프트 분리

## 5. 기술 스택

**플랫폼**: 모바일 앱 (iOS + Android)
**프론트엔드**: React Native / Expo (bunny-app 경험 재활용)
**백엔드**: Python FastAPI (bunny-app 패턴 재활용)
**LLM**: Ollama (로컬, RTX 5090)
**음성인식**: Whisper large-v3-turbo (bunny-app에 이미 있음, 나중에 사용)
**공개 접속**: Cloudflare Tunnel (bunny-app 설정 재활용)
**데이터베이스**: SQLite → PostgreSQL (나중에)
**알림**: Expo Push Notifications

**bunny-app에서 재활용**:
- Whisper 바이너리 + 모델
- Cloudflare Tunnel 설정
- FastAPI 구조 패턴
- React Native/Expo 구조 패턴
- OpenAI 호환 LLM 호출 패턴

## 6. 전체 아키텍처

```
📱 Expo 모바일 앱 (iOS/Android)
         ↕ HTTPS
🌐 Cloudflare Tunnel
         ↕
🖥️  FastAPI 백엔드 (Linux 홈서버)
   ├── 스케줄러 (APScheduler) — 루틴 시간에 푸시 알림 발송
   ├── 학습 이력 DB (SQLite)
   └── LLM 호출 모듈
         ↕
🤖 Ollama (localhost:11434)
         ↕
🧠 Llama 3.3 70B / Qwen 2.5 72B (RTX 5090 x2)
```
